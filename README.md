## 协作规范

本组织下所有仓库均采用统一的协作流程：

- **`main` 分支不接受直接推送**，所有修改须通过 Pull Request 合并
- 分支命名：`feature/描述`、`fix/描述`、`docs/描述`
- PR 须由主管（吴兰）或指定 reviewer review 后合并
- Commit message 格式：`type: 简要描述`（type 取值：`feat` / `fix` / `docs` / `chore` / `refactor`）

> 各仓库如有额外要求，以仓库内 README 为准。
